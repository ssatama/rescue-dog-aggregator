## Critical Issues Identified

### 1. 22,900 Broken Locale URLs (404s)

**Root Cause**: Old sitemap from October 2025 contained locale-prefixed URLs that were live briefly:

```
/en-GB/
/en-SK/about
/en-IT/dogs/snap-mixed-breed-1449
/en-PT/dogs/baloo-podenco-700
/en-FR/dogs/cooper-labrador-retriever-1587
/en-DE/dogs/mogly-pointer-883
/en-HU/dogs/lumi-mixed-breed-1407
/en-ES/dogs/estrella-mixed-breed-785
/en-RO/dogs/...
/en-FI/dogs/...
```

**Impact**:
- 92% of crawl budget wasted on dead URLs
- Google trust in site damaged
- Validation failed on 20.12.2025, rejected 6.1.2026

**Fix Required**: Add 301 redirects from `/en-XX/*` to `/*`

---

### 2. JSON-LD Schema in Wrong Tag

**Current (broken)**:
```html
<meta name="script:ld+json" content="{...}" />
```

**Should be**:
```html
<script type="application/ld+json">
  { "@context": "https://schema.org", ... }
</script>
```

**Location**: `frontend/src/app/dogs/[slug]/page.jsx` metadata generation

**Impact**: Google may not parse structured data correctly, missing rich results.

---

### 3. CSS/Static Files Being Indexed

**Issue**: `/_next/static/css/...` URLs appearing in Search Console

**Current robots.txt** blocks `/_next/data/` but not CSS files.

**Fix Required**: Block static assets from indexing.

---

### 4. 1,539 Pages "Indexed but not showing"

**Issue**: Google indexed these dog pages but chose not to show them in search results.

**Reason**: Google has too many similar pages and is selecting a subset to display.

**Potential Improvements**:
- Add more unique content to each page
- Add FAQ schema per dog
- Improve internal linking between related dogs

## Fixes to Implement

### Fix 1: 301 Redirects for Locale URLs

**File**: `frontend/next.config.js`

```javascript
async redirects() {
  return [
    // Redirect all old locale URLs to non-locale equivalents
    {
      source: '/en-:locale/:path*',
      destination: '/:path*',
      permanent: true, // 301
    },
    {
      source: '/en-:locale',
      destination: '/',
      permanent: true,
    },
  ];
}
```

### Fix 2: JSON-LD Schema Implementation

**File**: `frontend/src/app/dogs/[slug]/page.jsx`

Move schema from `metadata.other['script:ld+json']` to a proper `<script>` tag in the page component or use `generateMetadata` with the correct format.

**Option A**: Use Next.js built-in JSON-LD support
```jsx
export const metadata = {
  // ... other metadata
};

// In page component, render script tag directly
<script
  type="application/ld+json"
  dangerouslySetInnerHTML={{ __html: JSON.stringify(schema) }}
/>
```

**Option B**: Create a reusable JsonLd component
```jsx
// components/seo/JsonLd.tsx
export function JsonLd({ data }) {
  return (
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{ __html: JSON.stringify(data) }}
    />
  );
}
```

---

### Fix 3: Block Static Files from Indexing

**Option A**: Update `frontend/public/robots.txt`

```
# Block static assets from indexing
Disallow: /_next/static/css/
Disallow: /_next/static/chunks/
```

**Option B**: Add noindex header in `frontend/next.config.js`

```javascript
async headers() {
  return [
    {
      source: '/_next/static/:path*',
      headers: [
        { key: 'X-Robots-Tag', value: 'noindex' },
      ],
    },
  ];
}
```
